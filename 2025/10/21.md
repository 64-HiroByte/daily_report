## わかったこと
- オーバーロード関数では、関数シグネチャは上から順に評価され、最初に一致したものが採用される。したがって、引数が指定する範囲が狭い型を上にし、広くなる順で並べる必要がある。
- 「関数の代入可能性」は、右辺（代入する側）が左辺（受け取る側）の要求を満たせるかで判断される。
```ts
let x = (a: number) => 0;
let y = (b: number, s: string) => 0;

y = x; // ok!
x = y; // error
```
