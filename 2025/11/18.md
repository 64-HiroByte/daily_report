## わかったこと
- `seed.ts` において、import文は次のように記述しないとモジュールが見つからない
```ts
/** --- prisma/seed.ts --- */
// /client を末尾に追記した
import { PrismaClient, Prisma } from "../app/generated/prisma/client"; 

const prisma = new PrismaClient();

const userData: Prisma.UserCreateInput[] = [
  {
    ....
```
- Next.js 16で `seed` コマンドの記述は `prisoma.config.ts` に記述する。
```ts
/** --- prisma.config.ts --- */
export default defineConfig({
  schema: "prisma/schema.prisma",
  migrations: {
    path: "prisma/migrations",
    // ↓この1行を追加
    seed: `tsx prisma/seed.ts`,
  },
```
- `Route Handler` によるデータ取得ができない場合は、DBのurlパスをフルパスで記述する（`schema.prisma` と `prisma.config.ts` または`.env`）
- DB周りの設定を変えた場合は、`$npm prisma generate` を実行
- `useEffect` で無限ループが発生した場合は、依存配列を疑う
```ts
useEffect(() => {
  fetch("/api/sample")
    .then((res) => res.json())
    .then((data) => setUsers(data));
  // });  // ←無限ループ発生
  }, []);  // ←依存配列（空配列）を追加
```
